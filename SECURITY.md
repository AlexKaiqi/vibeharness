# Security Policy

## Reporting

Do not report sensitive security issues in public issues.

Until a public security contact is configured, report privately to the repository
owner. After the project moves to GitHub, enable GitHub private vulnerability
reporting and update this file with the advisory URL.

## Sensitive Data

VibeHarness episodes may contain commands, logs, screenshots, repository paths,
or user clarifications. Treat episode packages as potentially sensitive.

Before sharing an episode:

- remove secrets and credentials;
- remove customer or personal data;
- remove proprietary source code if not licensed for sharing;
- replace private URLs, hostnames, and tokens;
- verify that hidden checks are not leaked to worker-facing artifacts.

## Supported Versions

This project has no stable release yet. Security fixes should target the current
main branch until versioned releases exist.
