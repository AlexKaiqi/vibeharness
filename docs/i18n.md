# Documentation Localization

VibeHarness documentation supports English and Simplified Chinese.

## Policy

- English is the canonical source for now.
- Simplified Chinese files mirror English files with the `.zh-CN.md` suffix.
- Agent instruction files may remain English-only when they are read primarily
  by tools. Human-facing files should have Chinese counterparts.
- Every translated file must start with a language switcher:
  `English | 简体中文`.
- If English changes materially, update the matching Chinese file in the same
  pull request or mark it as stale in `.vibeharness/i18n.json`.

## File Naming

Examples:

- `README.md` -> `README.zh-CN.md`
- `docs/evaluation.md` -> `docs/evaluation.zh-CN.md`
- `benchmark/metrics.md` -> `benchmark/metrics.zh-CN.md`

Do not create language-specific folders for the repository README. GitHub shows
only one repository README, so the root README should link to the Chinese file.

For a future documentation website, use Docusaurus or MkDocs and generate a real
language switcher from the same source set.

## Translation Status

Translation state is tracked in `.vibeharness/i18n.json`.

Fields:

- `source`: canonical English file.
- `locale`: target locale.
- `target`: translated file.
- `status`: `current`, `stale`, `missing`, or `not_required`.
- `reviewed_at`: optional date used when a translation has been manually
  checked after the source changed.
- `notes`: optional maintenance notes.

## Maintenance Flow

1. Edit the English source.
2. Update the Chinese counterpart.
3. Run `make i18n-check`.
4. If translation cannot be completed in the same change, set status to `stale`
   and explain why in `notes`.

## What to Translate First

High-priority human-facing files:

- `README.md`
- `docs/landing_architecture.md`
- `docs/evaluation.md`
- `docs/agent_adapters.md`
- `docs/framework.md`
- `benchmark/README.md`
- `benchmark/metrics.md`
- `benchmark/scoring_rubric.md`

Lower-priority files:

- agent instruction files;
- paper draft;
- command templates;
- generated artifacts.
